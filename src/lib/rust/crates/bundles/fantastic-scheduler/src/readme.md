# scheduler — recurring tasks as an agent
Persists schedules to the store-relative `agents/<id>/schedules.json` + appends fires to `agents/<id>/history.jsonl` THROUGH a `file_bridge` named by its `file_bridge_id` meta (wire it to the `.fantastic` store — one store serves both records and sidecars). Owns no disk of its own: `boot`/`schedule`/`unschedule` **failfast with `file_bridge_id required`** until it's wired (no silent RAM/direct-write). Verbs: `reflect`, `boot` (idempotent; starts the tick loop; requires `file_bridge_id`), `shutdown`, `schedule {target, payload, interval_seconds}` (mints a `schedule_id`), `unschedule {schedule_id}`, `list`, `pause`/`resume {schedule_id?}`, `tick_now {schedule_id}`, `history {limit?, schedule_id?}`. On every fire, emits `schedule_fired` to its own inbox AND the target's inbox.

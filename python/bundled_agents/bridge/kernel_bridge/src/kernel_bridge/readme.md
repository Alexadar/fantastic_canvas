# kernel_bridge ‚Äî cross-kernel comms (WS-only, asymmetric)
A bridge agent opens a WS to the remote kernel's `web_ws` surface and ships raw `{type:"call", target, payload}` frames; the remote dispatches `kernel.send` exactly like any peer/WS client and replies over the same socket ‚Äî **no peer bridge needed**. Transports: memory (test backbone) / ws / ssh+ws. Streaming via `watch_remote` (`{type:"watch", src}` out, `{type:"event"}` back, re-emitted on the bridge's own inbox). Weak binding ‚Äî the remote is addressed by URL + path only, no shared types. Weak proxy ‚Äî local‚Üílocal stays direct `kernel.send`.

**Authorization** ‚Äî an optional per-leg `auth` policy gates inbound `call`s at one choke point (enforced on the receiver): `allow_all` (default ‚Äî absent ‚áí this, full symmetric duplex) | `deny_inbound` (one-way / hub‚Üíspoke push: refuse every inbound `call`, reply `{reason:"unauthorized"}`; the peer can't `call`/`reflect` back). The same seam serves cloud_bridge (both ride `bridge_core`).

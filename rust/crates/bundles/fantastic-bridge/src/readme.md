# ws_bridge / relay_connector ‚Äî cross-kernel comms (two io_bridge derivations)
TWO bundles share one engine (mirrors py's separate `ws_bridge` + `relay_connector`): `ws_bridge.tools` opens a WS to the remote kernel's `web_ws` surface (transports: memory (tests) / ws / ssh+ws); `relay_connector.tools` dials a relay-KERNEL router (`../fantastic_relay`) at `ws://<host>/<guid>` (group password in `X-Fantastic-Auth`, subprotocol `fantastic.relay.v1`) and reaches a `partner_guid` ‚Äî the relay routes by `target` and delivers peer‚Üípeer as one-way `{type:"event", source, payload}`. The connector TUNNELS bridge frames over that (`{type:"send", target:partner, payload:<frame>}` out, `event` in) so forward/reply correlation + symmetric serving work UNCHANGED; raw `read_stream` chunks ride as native binary WS frames (`[len|header|body]`, no base64). Each derivation only opens transports in its own family. A bridge ships raw `{type:"call", id, target, payload}` frames; the remote dispatches `kernel.send` exactly like any peer and replies over the same socket. Streaming via `watch_remote` (`{type:"watch", src}` out, `{type:"event"}` back, re-emitted on the bridge's own inbox). Weak binding ‚Äî the remote is addressed by URL/GUID only, no shared types. Weak proxy ‚Äî local‚Üílocal stays direct `kernel.send`. **Authorization** ‚Äî two symmetric, typed per-leg rules (enforced on the receiver): `ingress_rule` (inbound FILTER at one choke point) and `egress_rule` (outbound DECORATOR, stamps a credential on the frame envelope); a legacy `auth` shorthand sets both. Types: `deny_inbound` (**the DEFAULT ingress ‚Äî SEALED by default: an absent `ingress_rule` refuses every inbound `call`/`watch`/`emit`, reply `{reason:"unauthorized"}`; open with `ingress_rule=allow_all`**) / `allow_all` / `silent` (the default egress ‚Äî presents nothing) | `password` (kernel-GROUP shared secret ‚Äî ingress checks the envelope `auth_token` against the group token from an env var, default `FANTASTIC_GROUP_TOKEN`; egress presents it). These per-leg rules gate the tunneled bridge calls and are INDEPENDENT of the relay's own connection auth. Resolved by name from the `authorizer::ingress` / `authorizer::egress` registries (one module per rule). relay_connector record fields: `relay_url` ¬∑ `guid` ¬∑ `partner_guid` ¬∑ `relay_token` (X-Fantastic-Auth).
